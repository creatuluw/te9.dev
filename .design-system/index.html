Foundations

The atomic visual decisions that shape every component and page. Colors, type, spacing, elevation, and shape — consistent by default, reducing ambiguity and building trust through coherence.

Background Effects

Subtle texture overlays that add visual depth without distracting from content. Noise and scanlines reinforce the terminal aesthetic on a light background.

  • Noise overlay: A repeating SVG noise texture at 2–4% opacity layered over backgrounds. Adds warmth and grain without affecting readability.
  • Scanlines: Horizontal lines at 1px intervals with ~1% opacity. Creates a subtle CRT/terminal effect visible only on close inspection.
  • Opacity values: Noise at 0.02–0.04, scanlines at 0.01. Both use pointer-events: none and z-index: 1.

Rules

Non-negotiable constraints that keep the system consistent. Implementation standards, accessibility requirements, interaction states, and design principles.

Implementation

Eight mandatory rules that every contribution to this system must follow. They form the contract between design and engineering.

  • 1 Use vanilla CSS with custom properties in :root
  • 2 Semantic class names only — no utility strings like .flex.items-center
  • 3 Zero external CSS dependencies — no Tailwind, no Bootstrap, no frameworks
  • 4 Mobile-first responsive approach — base styles target smallest viewport
  • 5 Longhand padding for sections — never shorthand with .page-container
  • 6 60-30-10 color rule — 60% neutral, 30% content, 10% accent
  • 7 All animations must respect prefers-reduced-motion
  • 8 All interactive elements must have visible focus states

Accessibility

Every implementation must meet WCAG 2.1 Level AA minimum. Accessibility is not optional — it's a baseline requirement that ensures the interface works for everyone.

  • WCAG Level: 2.1 AA minimum
  • Contrast ratios: 4.5:1 for normal text, 3:1 for large text, 3:1 for UI components
  • Keyboard navigation: All interactive elements reachable and operable via keyboard
  • Focus visible: Outline rings on :focus-visible only, not on :focus
  • Skip link: Hidden until focused, links to #main-content
  • Reduced motion: All animations disabled when prefers-reduced-motion: reduce
  • Semantic HTML: Proper heading hierarchy, landmarks, ARIA where needed

  • Hover — transition: var(--duration-normal), visual brightness or color shift
  • Focus — outline: 2px solid var(--color-accent) with 2px offset, visible on keyboard only
  • Active/pressed — transform: scale(0.97) for tactile press feedback
  • Disabled — opacity: 0.5; cursor: not-allowed; pointer-events: none;

Spatial Hierarchy

Space communicates importance. Dense spacing signals related content; generous spacing signals separation. Our hierarchy moves from micro adjustments to spacious breathing room.

Micro (4px) — Inside components
Icon-to-label gaps, inline badge padding, checkbox-to-text spacing.
Tight (8px) — Between sibling elements
Buttons in a row, list items, form fields in a group, tags in a set.
Standard (16px) — Within sections
Card internal padding, label-to-input gap, paragraph spacing, component guts.
Comfortable (24px) — Between sections
Between form groups, card-to-card in a grid, header to content.
Spacious (48–80px) — Page sections
Between major page sections. The whitespace that gives the page room to breathe.

Design Principles

Twelve core principles that guide every visual and interaction decision. They provide a shared vocabulary for evaluating design choices and ensure every screen communicates with clarity and purpose.

Balance
Why: Stability & structure
Purpose: Distribute visual weight evenly
Align elements on an invisible axis; pair heavy text blocks with ample white space.
Contrast
Why: Visual interest & hierarchy
Purpose: Make elements distinct
Pair light and dark, large and small, bold and thin to guide the eye.
Emphasis
Why: Directs user attention
Purpose: Create a single focal point
Use size, color, or position to make CTAs unmissable — only one hero per screen.
Movement
Why: Controls content consumption
Purpose: Create a visual path
Use spacing, arrows, and progressive disclosure to lead users through content.
Pattern
Why: Predictability & reduced cognitive load
Purpose: Visual consistency
Repeat button styles, card layouts, and spacing rhythms across pages.
Proportion
Why: Communicates hierarchy
Purpose: Harmonious size relationships
Headings larger than body; icons proportional to text; containers sized to content.
Repetition
Why: Reinforces brand identity
Purpose: Unify through recurring elements
Reuse token values everywhere; never introduce one-off styles without updating the system.
Rhythm
Why: Predictable visual tempo
Purpose: Consistent spacing intervals
Apply the 8px grid strictly; equal gaps between cards; consistent section padding.
Unity
Why: Cohesive visual whole
Purpose: All elements belong together
Use shared tokens, consistent borders, and matching radii so every component feels unified.
Hierarchy
Why: Guides user understanding
Purpose: Establish clear levels of importance
Use size, weight, color, and position to create obvious visual priority at every level.
Variety
Why: Prevents monotony & visual fatigue
Purpose: Introduce controlled diversity
Alternate card orientations, mix image and text blocks, vary component density.
White Space
Why: Content needs room to breathe
Purpose: Separate & group by absence
Don't fill every pixel. Use spacing tokens to create visual breathing room — margins are not wasted space.

// Cross-fade transition
function transitionView(callback) {
  if (!document.startViewTransition) {
    callback();
    return;
  }
  const transition = document.startViewTransition(async () => {
    await callback();
  });
  transition.ready.then(() => {
    document.documentElement.animate(
      [{ opacity: 1 }, { opacity: 0 }, { opacity: 0 }, { opacity: 1 }],
      { duration: 300, easing: 'ease-in-out' }
    );
  });
}
