# Adding New Routes & Pages - Guide & Checklist

This guide outlines the process for adding new routes and pages to the application, following established patterns and best practices.

## Overview

When adding a new route/page, you need to:
1. Create the route directory and files
2. Implement page component with loading integration
3. Add navigation links (if applicable)
4. Follow styling and naming conventions
5. Handle data loading and error states

---

## File Structure

### Routes Location
All routes are created in: `src/routes/`

### Route Types

#### 1. **Static Routes**
Simple pages at fixed URLs:
```
src/routes/
├── about/
│   └── +page.svelte
├── contact/
│   └── +page.svelte
```

#### 2. **Dynamic Routes**
Pages with URL parameters:
```
src/routes/
├── cases/
│   └── [id]/
│       └── +page.svelte
├── users/
│   └── [userId]/
│       ├── +page.svelte
│       └── edit/
│           └── +page.svelte
```

#### 3. **Nested Layouts**
Pages sharing common layout:
```
src/routes/
├── admin/
│   ├── +layout.svelte
│   ├── +page.svelte
│   ├── users/
│   │   └── +page.svelte
│   └── settings/
│       └── +page.svelte
```

### Required Files
- `+page.svelte` - The page component (required)

### Optional Files
- `+layout.svelte` - Layout wrapper for nested routes
- `+page.ts` or `+page.server.ts` - Load data before rendering
- `+error.svelte` - Custom error page for this route
- `+layout.ts` or `+layout.server.ts` - Layout data loading

## Route Creation Checklist

### 1. Create Route Directory & Files

- [ ] **Directory naming**: Use kebab-case for static routes (e.g., `work/`, `user-settings/`)
- [ ] **Dynamic params**: Use `[paramName]` for URL parameters (e.g., `[id]/`, `[userId]/`)
- [ ] **Create `+page.svelte`**: Main page component file
- [ ] **Group related routes**: Use subdirectories for organization

#### Directory Structure Examples
```
Static route:
src/routes/dashboard/+page.svelte

Dynamic route:
src/routes/products/[id]/+page.svelte

Nested routes:
src/routes/settings/
├── +layout.svelte
├── +page.svelte
├── profile/
│   └── +page.svelte
└── preferences/
    └── +page.svelte
```

### 2. Implement Page Component (`+page.svelte`)

#### Basic Structure Pattern
```svelte
<script lang="ts">
	import { onMount } from 'svelte';
	import { page } from '$app/stores';
	import { goto } from '$app/navigation';
	import { navigationStore } from '$lib/stores/navigationStore';
	import * as yourService from '$lib/services/yourService';
	import type { YourType } from '$lib/types/yourType';
	import Button from '$lib/components/Button.svelte';
	import Spinner from '$lib/components/Spinner.svelte';

	let data = $state<YourType[]>([]);
	let loading = $state(true);
	let error = $state<string | null>(null);

	onMount(async () => {
		await loadData();
	});

	async function loadData() {
		loading = true;
		navigationStore.startPageLoading();
		try {
			const result = await yourService.getData();
			if (result.success) {
				data = result.value;
			} else {
				error = result.error.userMessage;
			}
		} catch (err) {
			error = 'Er is een fout opgetreden';
		} finally {
			loading = false;
			navigationStore.stopPageLoading();
		}
	}
</script>

<svelte:head>
	<title>Page Title - Business Process Management</title>
</svelte:head>

<div class="container mx-auto px-4 py-8">
	{#if loading}
		<div class="fixed inset-0 flex items-center justify-center">
			<Spinner size="xl" />
		</div>
	{:else if error}
		<div class="text-center py-12 text-red-600">
			{error}
		</div>
	{:else}
		<!-- Your page content -->
	{/if}
</div>
```

#### Required Elements Checklist

- [ ] **Import navigationStore**: `import { navigationStore } from '$lib/stores/navigationStore';`
- [ ] **Loading state**: `let loading = $state(true);`
- [ ] **Call startPageLoading**: `navigationStore.startPageLoading();` when loading starts
- [ ] **Call stopPageLoading**: `navigationStore.stopPageLoading();` when loading completes
- [ ] **Page title**: Use `<svelte:head>` with descriptive title
- [ ] **Container wrapper**: Use `container mx-auto px-4 py-8` for consistent spacing
- [ ] **Loading state UI**: Show loading indicator or use global spinner
- [ ] **Error handling**: Display user-friendly error messages
- [ ] **Empty state**: Handle cases when no data exists

### 3. Handle Dynamic Parameters

For routes with parameters (e.g., `/cases/[id]`):

```svelte
<script lang="ts">
	import { page } from '$app/stores';
	
	// Extract parameter
	const id = $derived(Number($page.params.id));
	
	// Validate parameter
	onMount(async () => {
		if (!id || isNaN(id)) {
			error = 'Ongeldig ID';
			return;
		}
		await loadData();
	});
	
	async function loadData() {
		loading = true;
		navigationStore.startPageLoading();
		const result = await service.getById(id);
		// ... handle result
		navigationStore.stopPageLoading();
	}
</script>
```

### 4. Data Loading Patterns

#### Pattern 1: Load on Mount
```svelte
onMount(async () => {
	await loadData();
});

async function loadData() {
	loading = true;
	navigationStore.startPageLoading();
	// ... fetch data
	navigationStore.stopPageLoading();
}
```

#### Pattern 2: Multiple Parallel Loads
```svelte
onMount(async () => {
	await Promise.all([loadUsers(), loadProjects(), loadStats()]);
});

async function loadUsers() {
	navigationStore.startPageLoading();
	// ... fetch users
	navigationStore.stopPageLoading();
}
```

#### Pattern 3: Reactive Loading (with effects)
```svelte
let filter = $state('all');

$effect(() => {
	loadData();
});

async function loadData() {
	navigationStore.startPageLoading();
	const result = await service.getData(filter);
	// ... handle result
	navigationStore.stopPageLoading();
}
```

### 5. Add Navigation Links

```svelte
<a
	href="/your-route"

```

#### Programmatic Navigation
```svelte
import { goto } from '$app/navigation';

async function handleClick() {
	await goto('/your-route');
}

// With query params
await goto('/your-route?filter=active');

// With state preservation
await goto('/your-route', { replaceState: true, noScroll: true });
```

### 6. Styling Guidelines

- [ ] **Container**: Use `container mx-auto px-4 py-8` for consistent page padding
- [ ] **Colors**: Follow zinc palette (`zinc-900`, `zinc-600`, `zinc-200`, etc.)
- [ ] **Cards**: Use `bg-white rounded-lg border border-zinc-200` pattern
- [ ] **Grids**: Use Tailwind grid classes (`grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-4`)
- [ ] **Responsive**: Mobile-first approach (use `md:`, `lg:` breakpoints)
- [ ] **Fonts**: `font-aspekta` for headings, `font-inter` for body text
- [ ] **Shadows**: `shadow-xs` for subtle elevation, `shadow-sm` on hover

## Common Patterns

### 1. List Page with Filters
```svelte
<script lang="ts">
	import { page } from '$app/stores';
	import SearchFilterPanel from '$lib/components/SearchFilterPanel.svelte';
	import SearchInput from '$lib/components/SearchInput.svelte';
	
	let searchQuery = $state('');
	let filterPanelOpen = $state(false);
	
	// Sync with URL params
	$effect(() => {
		const urlParams = $page.url.searchParams;
		searchQuery = urlParams.get('search') || '';
	});
	
	function updateUrlFromFilters() {
		const params = [];
		if (searchQuery) params.push(`search=${encodeURIComponent(searchQuery)}`);
		const queryString = params.length > 0 ? `?${params.join('&')}` : '';
		goto(`${$page.url.pathname}${queryString}`, { replaceState: true, noScroll: true });
	}
</script>

<SearchFilterPanel bind:open={filterPanelOpen} mode="button" />
<SearchFilterPanel bind:open={filterPanelOpen} mode="panel">
	<SearchInput
		bind:value={searchQuery}
		onchange={updateUrlFromFilters}
	/>
</SearchFilterPanel>
```

## Advanced Patterns

### Custom Layouts

Create `+layout.svelte` for shared layout:

```svelte
<!-- src/routes/admin/+layout.svelte -->
<script lang="ts">
	let { children } = $props();
</script>

<div class="admin-layout">
	<aside class="sidebar">
		<!-- Sidebar navigation -->
	</aside>
	<main class="content">
		{@render children?.()}
	</main>
</div>
```

### Protected Routes

Use `+page.ts` for authentication checks:

```typescript
// src/routes/admin/+page.ts
import { redirect } from '@sveltejs/kit';
import type { PageLoad } from './$types';

export const load: PageLoad = async ({ parent }) => {
	const { user } = await parent();
	
	if (!user || user.role !== 'admin') {
		throw redirect(307, '/login');
	}
	
	return {};
};
```

### Server-Side Data Loading

Use `+page.server.ts` for server-only data:

```typescript
// src/routes/reports/+page.server.ts
import type { PageServerLoad } from './$types';

export const load: PageServerLoad = async ({ locals }) => {
	const reports = await db.reports.findMany({
		where: { userId: locals.user.id }
	});
	
	return { reports };
};
```

## Common Pitfalls to Avoid

1. **Forgetting navigationStore integration**
   - Always call `startPageLoading()` and `stopPageLoading()`

2. **Not handling loading states**
   - Show loading indicator while data fetches

3. **Missing error boundaries**
   - Always handle and display errors gracefully

4. **Inconsistent styling**
   - Follow established patterns and zinc color palette

5. **Not using container wrapper**
   - Use `container mx-auto px-4 py-8` for consistent spacing

6. **Skipping empty states**
   - Always handle the case when there's no data

7. **Not validating dynamic params**
   - Check if `id` is valid before using it

8. **Forgetting page titles**
   - Always set `<svelte:head><title>` for SEO
